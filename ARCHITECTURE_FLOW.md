// Pseudocode of goal checking flow

func checkGoalCompletion(goal: Goal, usageMinutes: Int) {
    let isUnderLimit = usageMinutes <= goal.dailyLimitMinutes

    if isUnderLimit {
        // SUCCESS - Increment streak
        goal.currentStreak += 1
        goal.lastCompletedDate = today

        if goal.currentStreak > goal.longestStreak {
            goal.longestStreak = goal.currentStreak
        }

        // Check for milestone streaks
        if [7, 14, 30, 60, 100, 365].contains(goal.currentStreak) {
            sendStreakAchievedNotification(streak: goal.currentStreak)
        }

    } else {
        // FAILED - Reset streak, start recovery
        if goal.currentStreak > 0 {
            // Only create recovery if there was a streak to lose
            createStreakRecovery(
                for: goal.targetApp,
                previousStreak: goal.currentStreak
            )
        }

        goal.currentStreak = 0
        goal.lastBrokenDate = today
    }

    save(goal)
}

## ğŸ”„ Complete Flow Summary

1. **Setup Phase** (One-time)
   - User downloads app
   - Goes through onboarding
   - Grants Screen Time permission
   - Selects apps to track
   - Sets daily limits

2. **Tracking Phase** (Continuous)
   - iOS DeviceActivity framework tracks app usage in background
   - Extension writes data to Shared UserDefaults every ~15 minutes
   - App reads from Shared UserDefaults when needed

3. **Processing Phase** (Scheduled)
   - BackgroundMonitoringService runs periodically
   - Reads usage data from Shared UserDefaults
   - Updates SwiftData records
   - Checks goal completion
   - Triggers notifications if needed

4. **Display Phase** (On-demand)
   - User opens app
   - ViewModels load from SwiftData + Shared UserDefaults
   - Calculate progress and trends
   - Update UI with @Published properties
   - SwiftUI re-renders automatically

5. **End-of-Day Phase** (23:59)
   - Check all goals
   - Update streaks
   - Create recovery records if needed
   - Sync to CloudKit
   - Reset for next day
